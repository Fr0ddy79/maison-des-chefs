## Objective

Validate multi-chef inquiries so that:
1. Diners cannot submit inquiries with multiple services from the same chef
2. Unavailable services are rejected before lead creation
3. Guest count is validated per service
4. Related multi-chef inquiries are linked via a shared `multiInquiryId`

Additionally, display a "Multi-chef" badge on chef lead dashboard for grouped inquiries.

---

## Problem

`POST /api/multi-inquiry` currently accepts ANY array of service IDs without validating:
- That all services belong to different chefs
- That services are still available
- That guest count is valid per service

This creates invalid leads and a fragmented experience.

---

## Scope

### Backend (Backend Engineer)

**Step 1 — Validate chef diversity**
- Before creating leads, check that all selected services belong to different chefs
- If two services share the same `chefId`, reject with HTTP 400 and message: `"Please select chefs from different providers"`

**Step 2 — Validate service availability**
- Check each service has `available = true`
- If any service is unavailable, reject with HTTP 400 and message: `"One or more services are no longer available"`

**Step 3 — Validate guest count**
- For each service, check `minGuests <= guestCount <= maxGuests`
- Reject with HTTP 400 if validation fails, message: `"Guest count must be between {min} and {max} for {serviceName}"`

**Step 4 — Add schema fields**
```sql
ALTER TABLE chef_leads ADD COLUMN inquiry_type TEXT DEFAULT 'single';
ALTER TABLE chef_leads ADD COLUMN multi_inquiry_id TEXT;
```

**Step 5 — Link multi-chef leads**
- When multi-inquiry passes validation, generate a UUID as `multiInquiryId`
- Set `inquiryType = 'multi'` for all leads created from this inquiry
- Share the same `multiInquiryId` across all leads

**Step 6 — Update GET /api/chefs/:id/leads**
- Return `inquiryType` and `multiInquiryId` for each lead
- If a lead has `multiInquiryId`, include a `competingChefCount` field (count of other leads with same `multiInquiryId`)

### Frontend (Frontend Engineer)

**Step 7 — Chef Lead Dashboard badge**
- On lead cards/list items, show a "Multi-chef" badge when `inquiryType === 'multi'`
- Badge tooltip: "Part of a multi-chef inquiry"
- When clicking a multi-chef lead, show a subtle banner: "This diner also inquired with {N} other chefs"
