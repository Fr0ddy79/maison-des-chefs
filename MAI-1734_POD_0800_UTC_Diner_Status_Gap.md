## Feature Proposal: Diner Lead Status Page

### What It Is
A status page accessible via `token` link emailed after inquiry (already generated: `leads.accessToken` field exists). Shows real-time lead stage.

### Existing Infrastructure
- `leads.accessToken` — already generated at lead creation (MAI-805)
- `leads.status` — already tracked ("new", "responded", "expired", etc.)
- `leads.firstResponseAt` — SLA field already exists
- `leads.slaEscalated` — already tracks if lead was escalated

### Status Stages
```
[Inquiry Sent] → [Chef Received] → [Quote Ready] → [Booked/Declined]
      ✓                ?                    ?              ?
```

| Stage | Signal | Action |
|-------|--------|--------|
| Sent | ✅ Confirmed | "Chef responds within 24h" |
| No response (>12h) | ⏰ Stale indicator | "We'll nudge the chef" |
| Responded | 💰 Quote amount | Accept/decline links |
| Expired | 📭 Lead expired | "Browse other chefs" CTA |

### Scope

**In:**
- New route: `GET /lead-status?token=XXX`
- Status display: current stage + elapsed time
- Stale indicator with "We'll remind the chef" message
- Mobile-responsive, minimal UI

**Out:**
- Real-time WebSocket — poll every 60s
- Email integration (blocked on RESEND fix)
- Re-assignment flow

### Acceptance Criteria
- [ ] `GET /lead-status?token=XXX` accessible without auth
- [ ] Shows current lead status stage with elapsed time
- [ ] Stale leads (>12h no response) show "We'll nudge chef" message
- [ ] Responded leads show quote amount + accept/decline
- [ ] Mobile-responsive, no console errors

### Metrics
| Metric | Target |
|--------|--------|
| Diner abandonment (48h) | <40% from ~60% |
| "Remind Chef" click rate (stale) | >15% |
